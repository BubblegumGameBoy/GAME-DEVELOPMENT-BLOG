開発こぼれ話｜ブラウザで遊べるHTMLゲームを作って公開するまで

こんにちは、バブルガムです。普段はC#でゲームを作っているのですが、最近はブラウザでサクッと遊べるHTMLゲームも作って、この「Bubblegum Game」で公開しています。

なぜHTMLゲームなのか

いちばんの理由は、遊んでもらうまでのハードルが低いことです。ダウンロードもインストールも要らず、URLを開くだけ。PCでもスマホでも同じように動きます。腰を据えて遊ぶ作品はC#で作りつつ、「3分でクスッと笑える・スカッとする」軽いゲームはブラウザで――という住み分けが、自分にはしっくりきました。

使っているもの

使っている道具は、驚くほどシンプルです。

  • HTML / CSS / JavaScript … フレームワークなしの素のコード。1ファイルで完結することも多いです。
  • GitHub … ソースの管理と、後述する配信の両方に使っています。
  • GitHub Pages … 作ったHTMLをそのままWebサイトとして公開できる無料の仕組み。サーバー管理が不要なのが個人開発にぴったりです。

「ビルド環境を整える」みたいな前準備がほとんど要らないので、思いついたその日に形にできるのが気に入っています。

公開までの流れ

ざっくり、こんな順番で公開しています。

  1. ゲーム本体をHTML/JSで作る
  2. GitHubのリポジトリに置く
  3. GitHub Pagesを有効にすると、URLで遊べる状態になる
  4. このポータルサイトの各ゲームページから、そのゲームを埋め込み（iframe）で表示する

つまずいたところ

順調そうに見えて、地味に詰まったポイントもありました。

  • スマホ対応 … PCで快適でも、スマホだと文字が小さい・ボタンが押しにくい、はよくあります。最初から画面幅に応じて大きさを変える前提で作るのが正解でした。
  • 反映までのタイムラグ … GitHub Pagesは更新が反映されるまで少し時間がかかります。直したのに変わらない、と慌てず1〜2分待つのがコツです。
  • 「軽さ」を保つこと … 凝りだすと重くなりがち。ブラウザゲームの良さは手軽さなので、あえて足し算しすぎないようシンプルな設計を意識しています。

これから

アンダーテールが大好きで、プレイ中にちょっとした驚きを提供できるものを開発したいと考えています。私はネット黎明期の時代が好きで、それこそYoutubeに広告がなかった時代、２ｃｈのような掲示板がまだ主流だった昔のインターネットがすきです。
